# Structural Characteristics

## Single Reasoning Model

The framework deliberately avoids multiple reasoning systems.

Instead:

```
one node model
three analytical depths
```

This prevents fragmentation.

---

## Progressive Analytical Depth

| Tier       | Complexity | Use Case                              |
| ---------- | ---------- | ------------------------------------- |
| Core       | low        | explanation and basic advisory        |
| Advanced   | medium     | architecture and design reasoning     |
| Governance Extensions | contextual | governance controls when required by task conditions |

---

## Compatibility Across Tiers

Because tiers extend the same model:

* advanced analysis can degrade to core
* governance can attach without altering reasoning structure
* tools and documentation remain consistent

---

# Architectural Implication

This framework behaves like a progressively activated reasoning system whose active reasoning structure depends on the selected tier model and any attached governance extensions.

This design keeps the system:

* explainable
* extensible
* compatible with governance environments
